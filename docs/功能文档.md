## 1. 系统概述

**梦眠阁** 是一个 AI 驱动的智能睡眠管理平台，基于认知行为疗法 (CBT-I) 原理，帮助用户改善睡眠质量。系统提供睡眠记录、AI 分析、个性化任务、白噪音、社区互动、睡眠改善课程、游戏化激励等功能。

### 1.1 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | FastAPI (Python 3.11) |
| 数据库 | MySQL 8.0 + SQLAlchemy ORM |
| AI 引擎 | DeepSeek API (deepseek-chat) |
| 小程序（主力） | 微信小程序原生框架 |
| iOS / Android | Flutter 3.x（暂停维护，v3 MVP 已更新） |
| HarmonyOS | ArkTS + ArkUI（项目骨架） |
| 容器化 | Docker + Docker Compose |

### 1.2 应用入口

| 平台 | 入口 | 状态 |
|------|------|------|
| 微信小程序 | `miniprogram/app.js` | 主力，25 页 |
| Flutter (iOS/Android) | `flutter/lib/main.dart` | 暂停维护 |
| HarmonyOS | `harmony/entry/src/main/ets/pages/Index.ets` | 骨架就绪 |
| API 文档 | `/docs` (开发环境) | |

## 5. 核心功能特性

### 5.1 🔐 用户系统

**认证方式:**
- 用户名+密码注册/登录
- 微信小程序一键登录 (openid)
- JWT 双 Token 机制 (Access 30min + Refresh 7天)
- 密码强度验证 (字母+数字，最低6位)
- 密码重置 (邮箱Token)

**用户档案:**
- 基础信息: 昵称、头像
- 健康档案: 41个字段 (年龄、性别、身高体重、作息目标、睡眠问题、咖啡因、运动、压力、环境偏好等)
- 5步入门引导: 基本信息 → 睡眠目标 → 睡眠困扰 → 生活习惯 → 改善目标

### 5.2 📝 睡眠记录

**记录流程:**
1. 用户输入: 入睡时间 + 起床时间 + 质量评分(1-5) + 标签 + 备注
2. 自动计算: 睡眠时长 (精确到0.1h)
3. 智能评分: 综合时长+质量+标签计算 0-100 分
4. AI 反馈: DeepSeek 生成共情反馈 (CBT-I 原则，50字以内)

**评分算法:**
```
基础分 = 时长匹配度(0-50分) + 质量×6(0-30分) + 标签加权(-5~+20分)
最终分 = clamp(基础分, 0, 100)
时长匹配: 目标时长±1.5h → 50分; ±1h内 → 35分; 偏离越大分越低
```

**统计分析:**
- 平均时长/评分
- 作息规律度 (入睡时间标准差)
- 连续达标天数
- 睡眠效率 (实际睡眠/卧床时间)
- 睡眠债 (累计缺觉时长)
- 标签统计

### 5.3 🤖 AI 服务

**AI 模型:** DeepSeek-Chat (通过 REST API 调用)

| AI 功能 | 说明 |
|---------|------|
| **睡眠反馈** | 基于 CBT-I 的温柔共情反馈 (50字) |
| **任务生成** | 根据用户档案+近期数据生成4个个性化任务 |
| **音景设计** | AI 设计4通道白噪音混音方案 (13种声音类型) |
| **情感分析** | 分析用户文本的情感/焦虑水平 |
| **深度报告** | 5维度专业睡眠分析报告 |
| **质量预测** | 基于14天数据预测今晚睡眠质量 |
| **AI对话** | RAG增强的CBT-I睡眠教练 (21篇知识库) |
| **周报生成** | 结构化周报 + AI分析 + 行动建议 |

**RAG 知识检索:**
- 21篇睡眠科学文章 (预置在代码中)
- 6个分类: 睡眠科学、CBT-I疗法、睡眠卫生、放松技巧、特殊人群、睡眠误区、饮食与睡眠、压力与睡眠、科技与睡眠
- 关键词匹配 + 分类加权 → Top-K 检索

### 5.4 🎯 任务与游戏化

**每日任务:**
- AI 生成 4个个性化任务 (从18个预置任务中选择或AI创建)
- 时段分配: 早晨(光照/运动) → 下午(咖啡因/午睡) → 晚上(放松/环境)
- 每个任务 +5 积分

**徽章系统 (8种):**
🌱 初心者 | 📈 规律达人 | 🌴 冥想大师 | 🌙 早睡先锋 | 🌝 满月 | ♫ 白噪音达人 | ★ 百日修行 | 📘 睡眠博士

**等级系统:**
- 经验积累 → 等级提升
- 连续达标追踪
- 最高连续天数记录

### 5.5 📚 睡眠改善计划

**6个 CBT-I 改善计划:**

| 计划 | 周期 | 目标人群 |
|------|------|----------|
| 入睡困难改善 | 21天 | 入睡>30分钟 |
| 早醒问题改善 | 14天 | 凌晨醒后无法再睡 |
| 作息规律化 | 14天 | 作息不规律 |
| 深度睡眠提升 | 21天 | 睡眠浅、易醒 |
| 压力性失眠缓解 | 14天 | 焦虑型失眠 |
| 睡眠卫生基础 | 7天 | 所有人群/新手 |

每个计划包含周阶段划分 + 每日任务 + 打卡追踪 + 完成徽章。

## 8. 安全机制

| 层级 | 措施 |
|------|------|
| **传输** | HTTPS + HSTS (max-age=31536000) |
| **认证** | JWT (HS256) + Refresh Token 双Token机制 |
| **密码** | bcrypt 哈希 + 强度验证 (≥6位, 含字母+数字) |
| **令牌管理** | Token 黑名单 (登出失效), SHA256 哈希存储 |
| **限流** | IP+路径 60次/分钟; 登录 IP+用户名 5次/15分钟 |
| **XSS 防护** | 输入净化 (script标签/js协议/事件处理器/HTML标签) |
| **响应头** | X-Content-Type-Options, X-Frame-Options, X-XSS-Protection |
| **CORS** | 生产环境白名单 |
| **密钥** | 生产环境默认密钥告警 |
| **审计** | 审计日志 (登录/登出/密码修改等) |
| **缓存** | TTL 内存缓存 (可替换为 Redis) |

## 9. 部署架构

### 9.1 开发环境

```yaml
# docker-compose.yml
services:
  db:       mysql:8.0     → localhost:3307
  backend:  FastAPI       → localhost:8000
```

### 9.2 生产环境

```yaml
# docker-compose.prod.yml
services:
  db:       mysql:8.0     → 127.0.0.1:3307 (健康检查)
  backend:  FastAPI       → 127.0.0.1:8000 (健康检查+PRODUCTION=true)
  nginx:    nginx:alpine  → 0.0.0.0:80/443 (SSL+反向代理)
```

**生产配置要点:**
- MySQL 连接仅本地回环
- Nginx 反向代理 + SSL 证书
- 环境变量注入 (SECRET_KEY, DEEPSEEK_API_KEY, DB_PASSWORD)
- MySQL 自定义配置 (my.cnf)
- 容器健康检查 (db + backend)

### 9.3 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `DATABASE_URL` | MySQL连接串 | `mysql+pymysql://root:CHANGE-ME@localhost:3306/sleep_manager` |
| `SECRET_KEY` | JWT 签名密钥 | `change-this-in-production` ⚠️ |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Access Token 过期 | 30 分钟 |
| `REFRESH_TOKEN_EXPIRE_MINUTES` | Refresh Token 过期 | 10080 分钟 (7天) |
| `DEEPSEEK_API_KEY` | DeepSeek API Key | (必填) |
| `DEEPSEEK_BASE_URL` | DeepSeek API地址 | `https://api.deepseek.com` |
| `WECHAT_APPID` | 微信小程序 AppID | `wx551623c01821827f` |
| `WECHAT_SECRET` | 微信小程序 Secret | (必填) |
| `PRODUCTION` | 生产模式开关 | `false` |
